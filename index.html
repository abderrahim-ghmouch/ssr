Partie I : Client-Side Rendering

Définition & Mécanismes du CSR

Le Concept de SPA

Dans le CSR, le serveur ne fournit qu'un fichier HTML vide servant de "coquille". Le véritable moteur de l'application est un framework JavaScript (React, Vue, Angular) téléchargé par le client.

Le routage et le rendu des composants s'effectuent directement dans le navigateur, sans rechargement de page.

Cycle de Vie du CSR (Timeline)

1. Requête

Le serveur envoie une page HTML presque vide.

2. Assets

Téléchargement du Bundle JS (souvent lourd).

3. Hydratation

Exécution du JS pour monter le Virtual DOM.

4. Content

Appels API asynchrones pour les données finales.

Avantages & Défis du CSR

Points Forts

  • Expérience utilisateur fluide et réactive.
  • Charge serveur minimale (fichiers statiques).
  • Transitions instantanées entre les vues.

Points Faibles

  • First Contentful Paint (FCP) tardif.
  • Indexation SEO complexe pour les robots.
  • Dépendance totale à la puissance du CPU client.

Partie II : Server-Side Rendering

Comprendre le Rendu Serveur

Génération à la volée

Le SSR demande au serveur de construire le HTML final pour chaque requête. Les données sont récupérées en amont de l'envoi au client.

  • Contenu visible dès le premier octet.
  • Frameworks : Next.js, Nuxt, Laravel (Blade).
  • Optimisation naturelle pour les crawlers.

Cycle de Vie du SSR (Timeline)

1. Appel

Le client demande une URL spécifique.

2. Compute

Le serveur fetch les données et rend le HTML.

3. Stream

Le HTML complet est envoyé au navigateur.

4. Interact

Le JS "réveille" les composants (Hydratation).

Avantages & Défis du SSR

Atouts Majeurs

  • Visibilité immédiate (FCP ultra-rapide).
  • Parfait pour le SEO et réseaux sociaux.
  • Meilleure sécurité pour les API privées.

Inconvénients

  • Charge de calcul élevée côté serveur.
  • Time to First Byte (TTFB) plus élevé.
  • Gestion d'état complexe entre Server/Client.

Stratégies Hybrides Modernes

SSG & ISR

  • SSG : Génération Statique au build.
  • ISR : Mise à jour incrémentale en background.

Architecture en Îlots

  • HTML statique par défaut.
  • JS uniquement sur les zones interactives.
  • Exemple : Astro, Qwik.
