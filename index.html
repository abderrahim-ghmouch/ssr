Client-Side Rendering

2. Qu'est-ce que le CSR ?

Rendu Côté Client

Le rendu côté client (CSR) est une technique moderne où la responsabilité du rendu de la page est confiée au navigateur.

Le navigateur reçoit un squelette HTML minimal et un volumineux fichier JavaScript qui construit l'interface dynamiquement.

4. Avantages & Limites du CSR

Avantages

Expérience fluide (SPA), transitions instantanées, moins de charge serveur.

Limites

Chargement initial lent, défis SEO, dépendance matérielle client.

Partie II : SSR

Server-Side Rendering

5. Qu'est-ce que le SSR ?

Rendu Côté Serveur

Le serveur génère la structure HTML complète avant l'envoi.

  • Génération complète sur serveur.
  • Données pré-remplies.
  • Prêt à être vu instantanément.

6. Fonctionnement du SSR

1. Requête

Navigateur demande.

2. Rendu

Serveur compile HTML.

3. Envoi

HTML complet affiché.

4. Hydratation

JS attache interactivité.

7. Avantages & Limites du SSR

Avantages

SEO supérieur, affichage initial ultra-rapide, indépendant du hardware client.

Limites

Requêtes serveur fréquentes, coûts infra élevés, TTFB plus long.

8. Performance & SEO

Métriques

FCP : SSR gagne. TTI : CSR brille après loading. TTFB : CSR plus rapide via CDN.

Le SEO

SSR offre une sortie lisible immédiate. Les balises OG nécessitent le HTML initial.
